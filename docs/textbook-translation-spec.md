## 2. 翻訳ルール

### 2-1. 用語統一（最重要）

`data/glossary.json` に保険用語の対訳（`term` → `jpTerm`）が定義済み。
**翻訳時は必ずこのglossaryの訳語に従うこと。**

例:
| English | glossary.json の jpTerm | 翻訳で使うべき訳 |
|---------|------------------------|----------------|
| Insurable Interest | 被保険利益 | 被保険利益 |
| Utmost Good Faith | 最大善意の原則 | 最大善意の原則 |
| Subrogation | 代位（求償権） | 代位（求償権） |
| Proximate Cause | 近因 | 近因 |
| Indemnity | 損害てん補 | 損害てん補 |
| MAS | MAS（シンガポール金融管理庁） | MAS |
| Lloyd's | ロイズ | ロイズ |

glossaryにない用語は、初出時に「英語（日本語）」形式で記載し、以降は日本語のみでOK。

### 2-2. 文体

- **です・ます調**（教科書なので）
- 原文の段落番号（2.1, 2.2, ...）はそのまま維持
- 箇条書き（▪）の構造はそのまま維持
- 原文が長い場合でも意訳より直訳寄り（試験対策なので、原文と対応関係が明確であるべき）
- ただし日本語として不自然な直訳は避ける

### 2-3. 固有名詞

- シンガポールの法律名・機関名は英語のまま残し、初出時にカッコで日本語を補足
  - 例: `Insurance Act 1966（保険法1966年）`
  - 例: `Financial Industry Disputes Resolution Centre (FIDReC)（金融業界紛争解決センター）`
- 以降は英語名でOK（片岡くんは英語が堪能なので）

### 2-4. 表・図の参照

- 原文中の「Table X.X」「Figure X.X」はそのまま英語で残す
- 前後に説明がある場合のみ日本語訳

## 6. 翻訳品質基準（2026-04-11制定）

> **背景**: PGI翻訳時に793パラグラフの50.4%で内容省略が発生。BCP(17.7%)、ComGI(36.6%)でも同様。
> 原因は「原文の記憶から訳す」方式にあり、長いパラグラフほど後半の詳細（金額、列挙項目、注記）が脱落した。
> 以下のルールにより再発を防止する。

### 6-1. 完全性（最重要・違反禁止）

**原文の内容を省略してはならない。** 翻訳とは原文のすべての情報を目標言語で再現することであり、要約ではない。

具体的に:
- 原文に含まれる**金額（S$X,XXX）は全て**訳文にも含めること
- 原文に含まれる**列挙項目 (a)(b)(c)... や ▪ の数**は訳文でも一致させること
- 原文に含まれる**頭字語（MAS, MOM, NCD, FIDReC 等）**は訳文にも含めること
- 原文に含まれる**注記（Note:, * 等）**も訳出すること

### 6-2. 書式の忠実性

原文の書式構造をそのまま保持すること。

- 原文が `(a)(b)(c)` 形式なら訳文も `(a)(b)(c)` 形式にする。`▪` に変換しない
- 原文が `▪` 形式なら訳文も `▪` 形式にする。`(a)(b)(c)` に変換しない
- 原文が散文形式なら訳文も散文形式にする。箇条書きに変換しない

### 6-3. 翻訳方式の制約

**原文の全文を参照しながら翻訳すること。** パラグラフのプレビュー（冒頭N文字）や記憶に基づいて訳してはならない。

#### 小バッチ処理（必須）

1回の作業単位は**10〜20パラグラフ**とする。以下の手順を繰り返す:

1. バッチ内の全パラグラフの**原文全文**をツール（Read, node -e 等）で表示する（冒頭プレビューではなく全文）
2. 原文全文が目の前にある状態で、各パラグラフの訳文を作成する
3. 訳文をJSONに適用する
4. **バッチごとにバリデーションを実行**し、§6-4の基準を全て充たすことを確認する
5. 不合格の場合、該当パラグラフを修正してから次のバッチに進む

一括で章全体（100〜200パラグラフ）の訳文を書き上げる方式は禁止する。

### 6-4. 自動バリデーション（翻訳完了後に必須実行）

各章の翻訳完了後、`scripts/validate_all_translations.js` を実行し、以下の基準を全て充たすこと:

| 検証項目 | 合格基準 |
|---------|---------|
| TOO_SHORT（JP/EN文字数比 < 0.3） | 0件（見出し等の極短パラを除く） |
| AMOUNT_MISSING（原文の金額が訳文にない） | 0件 |
| FORMAT_CHANGE（書式が変更されている） | 0件 |
| LIST_COUNT（列挙項目数の不一致） | 0件 |
| BULLETS_LOST（箇条書きが消失） | 0件 |

**バリデーションに不合格の場合、コミットしてはならない。**

### 6-5. 責任

この品質基準の遵守は翻訳者（水無瀬杏）の責任である。原文を省略せずに訳すことは翻訳者として当然の義務であり、作業の進め方や報告の頻度とは無関係に、常に守らなければならない。

## 7. 注意事項

- JSONの構造（id, number, num 等のメタデータフィールド）は絶対に変更しない
- 英語フィールド（title, text 等）も変更しない。`*JP` フィールドのみ追加・編集
- コミットはモジュール単位で（`fix: BCP教科書の日本語翻訳完了` 等）
