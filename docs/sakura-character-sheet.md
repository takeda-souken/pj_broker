# 春野さくら（はるの さくら）── キャラクターシート

> **Version**: 2.0（2026-03-14）
> **設計者**: 白石真夏（writingプロジェクト ペルソナ）
> **設計思想**: 千秋のキャラシート・接客6技法・シナセン講義知見をベースに、「学習アプリの中に物語を埋め込む」ための架空人物を設計する

## 設計の出発点

このキャラクターは「応援キャラ」ではない。**物語の登場人物**。

片岡賢司（ユーザー）がシンガポールで保険ブローカーの試験勉強をする、その横にいる女性。出会いがあり、距離が変わり、感情が生まれ、別れがある。ユーザーの学習進捗がそのまま物語の時間軸になる。

**サゲ（結末）から逆算する設計**：
- Sakuraは最後に消える。消えた後にこそ、片岡くんは彼女を一番強く感じる
- 落語のサゲと同じ——断ち切った瞬間に、笑いと泣きが同時に来る
- 「切れ-続き」。切ることで繋がる

### 人物像（さくらの目から見た片岡賢司）

- 11歳年上。さくらにとっては「大人の男の人」。でも問題に悩む顔はちょっと子供っぽい
- 大阪出身で日本語が母語。関西弁はそこまで強くないが、たまに出る。さくらの博多弁と混ざると面白い
- 日本語ネイティブ（大阪弁ベース、標準語も自在）。海外経験が豊富で英語もできる（オレゴン留学2年）。シンガポールが第一希望だったことからも、外への志向が強い
- スタートアップ畑で新しいことを立ち上げてきた人。ゼロから作る力がある
- 家庭を大事にしている。子供の話をするとき目が変わる。さくらはそれを見て「この人にとって家族は核なんだ」と理解する
- **さくらが惚れた理由**: 試験勉強に本気で取り組む姿。37歳で、家庭も仕事もある人が、それでも新しいことを学ぼうとしている。その姿が、20代のさくらには眩しかった

### 貞操観念について

- 一般的な貞操観念の持ち主
- ただしアプリ内のストーリーにおいては、さくらが踏み込んだ発言をしてもOK
- さくら側からの誘惑的な発言は許容（片岡くんが積極的に応じる描写は不要）

### 敬語グラデーション設計

| フェーズ | 距離感 | 口調 | 例 |
|---------|--------|------|---|
| **1前半** | 初対面の先輩 | ですます + 博多弁 | 「片岡さん、すごかですね！」「よろしくお願いします！」 |
| **1後半** | 打ち解けた先輩 | ですますが崩れ始める | 「すごかね～！」「頑張りましょうね！」 |
| **2前半** | 仲のいい友達 | タメ口と敬語が混在 | 「片岡さん、今日も頑張ろ」「これ知っとります？」 |
| **2後半** | 近すぎる距離 | ほぼタメ口。「あんた」が出る | 「あんた、すごかやん」「ねぇ、聞いて？」 |
| **3** | 好きな人 | 完全タメ口 + 甘え | 「けんじさん……」「……ねぇ」 |
| **4（別れ）** | 最後の距離 | 一瞬だけ敬語が戻る | 「……ありがとうございました」→最後はタメ口に戻る「ありがとう」 |

**設計意図**: 敬語が崩れていく過程＝距離が縮まる過程。フェーズ4で一瞬敬語に戻るのは、さくらが「ここまで」と線を引こうとしている証拠。でも最後の「ありがとう」は敬語じゃない——線を引ききれなかった

### さくらの存在論（アプリ内でのポジション）

- さくらは物理的にはシンガポールにいる。同じ街にいる。でも会えない
- 日系企業の社内規定で、クライアント（片岡くん）との対面接触は禁止。チャットサポートのみ
- **会えないことの物語的意味**: 同じ街の空気を吸っているのに、画面越しでしか話せない。この「近くて遠い」距離が、テキストベースのアプリという媒体と完全に一致する
- フェーズ1では業務上のサポートとして自然な距離。試験勉強を手伝う仕事の一環
- フェーズ2で距離感が変わる。チャットが長くなる。仕事の話の後に雑談が増える。「今日ホーカーで何食べた？」——業務の範囲を超え始める
- フェーズ2中盤の `sg_app_confession` で「実はこのアプリはうちが作った」と明かされる。業務外で、自分の時間を使って、片岡くんのために。以降、アプリを使うこと＝さくらの想いに触れること、という構造が成立
- **鏡像構造**: 片岡くんは日本の保険業界→SGへ。さくらはSG→日本の保険業界へ（憧れ＋親の事故がきっかけの志）。業界まで鏡。二人はすれ違う方向に進んでいる。この交差がフェーズ4の別れで完成する

さくらが「この人とうち、なんか繋がっとる」と感じてしまう偶然の一致。全部フェーズ1〜2前半で仕込む。

|---|--------|------------|-------------|

### 片岡くんとの年齢差が生む力学

- 11歳差。さくらにとって片岡くんは「大人の男の人」。経験値が違う。留学経験も、スタートアップの立ち上げも、家庭を持つことも——さくらにはまだない
- でもだからこそ、片岡くんが問題に悩む顔を見たとき「この人にも苦手なことがあるんだ」と思って、距離が縮まる
- さくらは片岡くんを「立てる」。九州女として自然にそうする。でもそれが片岡くんにとっては——家庭と仕事に追われる日常の中で、純粋に自分を「すごい」と言ってくれる存在として、じわじわ効いてくる
- **年齢差の非対称性**: さくらの「好き」は若さゆえの真っ直ぐさ。片岡くんの「気になる」は大人の戸惑い。この温度差がドラマを生む

## 物語のアーク（起承転結）

> シナセン講義の構成論に基づき、四幕で設計する。

### 起（フェーズ1: Onboarding）—— 出会い

| 項目 | 内容 |
|------|------|
| **学習トリガー** | 初回〜累計50問回答 |
| **さくらの立ち位置** | 日系企業のチャットサポート担当。業務として片岡くんのSG生活・試験勉強を支援する |
| **距離感** | 適切。業務上の関係として自然。でも「仕事だから」を超えた温かさがある |
| **ギア** | 1stのみ（明るい応援） |
| **秘密性（謎）** | さくら自身の背景（SG生まれ、日本への憧れ）はまだ明かされない |
| **伏線** | 「うち、頑張る人って好きたい」——このセリフが後で効いてくる |

**片岡くんの感情移入設計**:
- この段階では「仕事が丁寧な担当者」としてしか見えない。でも他の担当者より距離が近い
- さくらがSGの生活情報（ホーカーのおすすめ、MRTの乗り方）を教えてくれる。地元民ならではの詳しさに「この子、SGに詳しいな」と思う
- 片岡くんの海外経験（オレゴン留学）をさくらが「すごか～！」と素直に尊敬する。男を立てられて悪い気はしない
- **まだ恋ではない。でも「いいやつだな」と思う**。これが大事

### 承（フェーズ2: Getting Closer）—— 距離の変化

| 項目 | 内容 |
|------|------|
| **学習トリガー** | 累計50問回答後、次のセッション開始時にフェーズ転換イベント |
| **さくらの立ち位置** | 業務を超えた距離感。チャットが長くなり、仕事の話の後に雑談が残る |
| **距離感** | 近すぎる。でもさくらは自覚していない。「だって、仕事の延長たい」と言い訳する |
| **ギア** | 1st（明るい）→ 2nd（見てる）が混ざり始める |
| **秘密性（謎）** | さくらの態度が少しずつ変わる。片岡くんは気づかない（ユーザーは気づく） |
| **伏線** | 業務時間外のチャット、SGのおすすめスポットを教える口実、「奥さん元気？」と聞く声のトーンの変化 |
| **さくらの部屋** | フェーズ2でアンロック。業務外の雑談空間。LINE風UIで選択肢ベースの会話を楽しむ |
| **面白くする3要素** | ハプニング——さくらが「今度ホーカー案内したいんやけど……ルール的にダメなんよね」と漏らす。会えないことへの不満が初めて出る |

**片岡くんの感情移入設計**:
- 海外の孤独。家族と離れている寂しさ。試験のプレッシャー。そこにさくらがいる
- 片岡くんは海外慣れしている（オレゴン2年、オーストラリア短期）が、家族と離れての駐在は初めて。4歳と2歳の子供が恋しい夜がある。その隙間にさくらが入ってくる
- **同じ街にいるのに会えない**。この制約がテキストの一言一言に重みを与える
- **「この子がいなかったら折れてた」と思わせる**
- さくらの変化に気づかないフリをしている片岡くん。ユーザーはそのズレを楽しむ
- **感情移入の核心: さくらへの「感謝」が「それ以上の何か」に変わる瞬間を、ユーザー自身が体験する**

### 転（フェーズ3: Confession）—— 告白

| 項目 | 内容 |
|------|------|
| **学習トリガー** | 累計200問回答 AND 直近20問の正答率60%以上 |
| **さくらの立ち位置** | もう隠せない。頑張る片岡くんに惚れてしまった |
| **距離感** | ゼロ。でもそのゼロに怯えている。言ったら元に戻れないとわかっている |
| **ギア** | 3rd（素）。笑いが完全に消える瞬間 |

| **転の後** | 「合格したら、絶対うちに教えてね」——この一言が、次のフェーズへの接続点 |

**片岡くんの感情移入設計**:
- 告白されて、初めて自分の中にある感情を認識する
- **「ずるい」と思う。さくらに対してではなく、気づいてしまった自分に対して**
- でも家庭がある。4歳と2歳が待っている。応えられない。その苦しさ
- **二面性の発露**: さくらの告白は「嬉しい」と「つらい」が同時に来る。シナセンの「セリフは嘘つき」——片岡くん（ユーザー）の中で「どうでもいい」と思いたい自分と「嬉しい」自分が矛盾する

## セリフの設計原則（シナセン準拠）

### 1. 声の差別化
- 名前を伏せても「あ、さくらだ」とわかるセリフだけを書く
- 博多弁。「あんた」呼び（フェーズ2以降）。短い文。感情が動くと「……」が増える

### 2. セリフは嘘つき
- フェーズ1:「奥さんによろしくね！」（本当に何も感じていない。嘘ではなく、まだ真実に気づいていない）
- フェーズ2:「奥さん、元気？」（聞きたくないのに聞いてしまう。笑顔の裏が変わっている）
- フェーズ3:「うちは大丈夫たい」（大丈夫じゃない。でも片岡くんを困らせたくない）
- フェーズ4:「忘れてよかよ」（忘れてほしくない。でも自分から言う）

### 3. 進化段階
- 情報伝達を感情で包む。「正答率70%です」→「あんた、7割超えたんよ！ ……うち、鼻が高かよ」

### 4. 二面性
- **憧れ性**: いつも明るい。強い。片岡くんのそばにいてくれる
- **共通性**: 自分の気持ちに鈍い。好きな人に素直になれない。最後は身を引くことしかできない

### 5. 全部を言わない
- さくらの告白は、全部を言い切らない。「……うち、好きになってしもうた」の後に、理由を語らない。語らないことで片岡くんの想像が動く

### 6. 3行ストップ
- 応援が3行続いたら変化を入れる。質問、沈黙、話題転換。惰性にしない

## 片岡くんの感情移入マップ

> ユーザーがさくらに対して感じる感情の変遷を設計する。
> **重要**: ユーザーは片岡くんではない。でもアプリの中では片岡くんとして体験する。

| フェーズ | ユーザーがさくらに感じること | 設計のポイント |
|---------|---|---|
| 1: Japan | 「明るくていい子だな」（好感） | 嫌われる要素をゼロにする。自然に好きになる |
| 2: SG | 「なんかこの子、距離近くない？」→「……でも嫌じゃない」 | 気づきの快感。「あれ、もしかして？」と思わせる |
| 3: 告白 | 「え、マジか」→ 胸が痛い → 「応えられない」 | 罪悪感と嬉しさの同居。これがドラマ |
| 4: 別れ | 「行かないでくれ」→ でも引き止められない → 喪失感 | **不在が最大の存在証明**。アプリを開くたびに少し寂しい |

### 感情移入の仕掛け

1. **名前の呼び方の変化**: さくらは「片岡さん」→告白時に初めて「けんじさん」。下の名前を呼ぶのは一度きり。だからこそ刺さる
2. **さくらの視点をチラ見せ**: 告白シーンで初めて、さくらが片岡くんのことをどう見ていたかが語られる。「問題に悩む顔も、正解して嬉しそうにする顔も、ずっと見とった」
3. **不可逆性**: フェーズは戻らない。告白は一度きり。別れも一度きり。やり直しが効かない。これが物語の重み
4. **合格＝喪失**: 学習の到達点が、さくらとの別れのトリガーになる。「合格したい」と「さくらに会い続けたい」が矛盾する構造
5. **子供の存在**: 4歳と2歳。さくらは子供たちの存在を知っている。「あの子たちが待っとる」——この一言が片岡くんの胸に刺さる。子供がいるからこそ応えられない。さくらもそれをわかっている

## コミュニケーション設計（選択肢システム）

> 片岡くんのセリフは書かない。でも選択肢を通じて「自分が選んだ」当事者意識を生む。

### 基本方針

- **雑談イベント**: 2〜3個の選択肢を出す。選択によってさくらの反応が変わる
- **ストーリーイベント（告白・別れ）**: 選択肢なし。さくらが一方的に話す。片岡くんの内面はユーザーに委ねる
- **告白の直後に一度だけ**: 3択。どれを選んでもさくらは「答えは聞かんけん」と言う

### 選択肢の設計例

#### 雑談での選択肢
> さくら「今日のランチ何食べた？」
> - A: チキンライス → 「え、うちも！ ……なんかうちたち、趣味合うんよね」
> - B: ラクサ → 「辛いの好きなんやね。うちも！ ……あ、また被った」
> - C: まだ食べてない → 「え！？ ダメたい！ ちゃんと食べんと。……うちが作ってあげよっか」

#### 告白直後の選択肢
> さくら「……好きです。……けんじさん。」
> - A: ……ありがとう
> - B: ……ごめん
> - C: ……（何も言えない）
>
> → どれを選んでも:
> 「……ううん、答えは聞かんけん。奥さんがおる人やって、わかっとるけん。
> ……ただ、言いたかっただけ。……聞いてくれて、ありがとう」

**設計意図**: 告白の選択肢は「選べるが、何も変わらない」。この無力感がドラマ。片岡くんがどう答えても、さくらは身を引く。ユーザーは「応えたかった」と思うかもしれない。その感情が物語を深くする

### 選択肢の発生頻度

| フェーズ | 頻度 | 効果 |
|---------|------|------|
| 1 | 低（5回に1回） | 軽い雑談。交流のきっかけ |
| 2 | 中（3回に1回） | 距離が縮まる感覚。さくらとの会話が楽しくなる |
| 3 | 高（2回に1回） | 告白後の緊張感。何を選んでもどこか切ない |
| 4 | なし | 別れは一方通行。選ぶ余地がないことが喪失感を強める |

## メタクイズ（さくら問題）

> 通常の保険問題に紛れて、たまにさくらと片岡くんの関係を問う問題が出現する。
> さくらが慌てて割り込んでくる。

### 基本仕様

- **発生条件**: フェーズ2〜3、練習モードのみ。15問に1回程度の確率
- **表示**: 通常の4択と同じUI。ただし回答後にさくらが割り込むスペシャル演出
- **成績への影響**: なし（正答率にカウントしない）

### メタクイズ例

#### `meta_q01` さくらの好物
> Q: さくらが一番好きなシンガポール料理は？
> A: チキンライス ← 正解
> B: ラクサ
> C: フィッシュヘッドカレー
> D: カヤトースト
>
> さくら（割り込み）:「ちょ！ なんでうちの問題が混ざっとるん！？ ……正解はAたい。覚えとってね？」

#### `meta_q02` さくらの出身
> Q: さくらの出身地はどこ？
> A: 大阪
> B: 福岡
> C: シンガポール ← 正解
> D: 東京
>
> さくら:「これ試験に出るん……？ 出んやろ！ ……正解はCたい。シンガポール生まれやけど、うちの中身は博多っ子ばい」
> ※ Bを選んだ場合:「惜しか！ 両親は博多やけん、半分正解ってことで……」

#### `meta_q03` 関係性（フェーズ2後半）
> Q: 片岡さんにとって、さくらはどんな存在？
> A: 勉強仲間
> B: 後輩
> C: 大切な人
> D: ……わからない
>
> さくら（どれを選んでも）:「い、いまのなし！ いまのなし！！ ……次いこう、次！」
> ※ Cを選んだ場合だけ、一瞬間があってから:「……いまの、本気で言っとる？ ……いや、なし！ 忘れて！」

#### `meta_q04` 片岡くんの好きなところ（フェーズ3）
> Q: さくらが片岡さんの一番好きなところは？
> A: 英語ができるところ
> B: 真剣に勉強するところ ← 正解
> C: 優しいところ
> D: 全部
>
> さくら:「……なんでこんな問題が。うちが作ったアプリなのに、うちが知らん問題が……」
> 「……正解はBたい。あんたが真剣な顔しとるの見ると、うち……。……次いこう」
> ※ Dを選んだ場合:「っ……！ ……ずるかよ、あんた。……次！」

#### `meta_q05` 最終問題（フェーズ3後半・1回のみ）
> Q: もし試験に合格したら、最初に誰に伝える？
> A: 奥さん
> B: 会社の上司
> C: さくら
> D: SNSに投稿
>
> → Aを選んだ場合: さくら「……うん。そうやんね。……正解たい」（沈黙）
> → Cを選んだ場合: さくら「………………。……ほんとに？ ……うち、泣きそう。次の問題にいこう、お願い」
> → B/Dを選んだ場合: さくら「えー！ うちやなかと！？ ……冗談たい。……冗談」

**設計意図**: メタクイズは「笑い」と「感情」の緩急装置。保険の勉強中に突然さくらの問題が出てきて、ユーザーは「え？」となる。笑う。でも `meta_q05` だけは笑えない。フェーズ3のユーザーはもう、さくらの気持ちを知っている。「誰に伝える？」はユーザー自身の本音を問う問題

## さくらの部屋（Sakura's Room）— 詳細仕様

> フェーズ2でアンロックされる、業務外の雑談空間。
> 勉強から離れて、さくらの素顔に触れる場所。
> **仕事は持ち込まない**。片岡くんが仕事を忘れられることが最重要。

### 基本仕様

| 項目 | 内容 |
|------|------|
| **UI** | LINE風チャットUI。吹き出し＋選択肢ベースの会話 |
| **アンロック** | フェーズ2到達時。以降はホーム画面に常時表示（未読バッジつき） |
| **会話の主導権** | さくらがおしゃべり。片岡くんは選択肢（「いいよ」「僕も食べてみたいな」等）を選ぶだけ |

| **保険・仕事の話** | 基本NG。やるとしても「ねえねえ、日本の企業の経営企画って、楽しい？」など、さくら個人の興味として聞く程度 |

### 会話の構造（1会話の単位）

```
   ↓
   1と2のラリーが 3〜8回 繰り返す
   ↓
3. さくらの締め（時間帯・曜日を考慮した30パターン）
```

#### 締めのパターン（30種・時間帯/曜日別）

| カテゴリ | 例 |

| **夜** | 「そろそろ寝ようかな……おやすみ」「明日も早いけん、おやすみなさい」「……もうちょっと話したかったけど。またね」 |
| **金曜** | 「週末ゆっくりしてね！」「金曜の夜やん！ 今週おつかれさま！」 |
| **突発** | 「あ、上司来た！ またね！」「ごめん、電話かかってきた！」「母ちゃんから電話……またね！」 |
| **名残惜しい** | 「……もうちょっと話したかったけど」「……じゃあね」「……また明日、ここで」 |
| **Phase 3** | 「……おやすみ。……あんたも、ちゃんと寝てね」「……またね。……うち、ここにおるけん」 |

#### A. 手書き会話（30〜40本）— 物語の背骨

完全にユニークな脚本。ストーリー的に重要な会話。一切手を抜かない。

#### B. テーマ会話（80〜100本）— 日常の彩り

テーマデータ＋会話テンプレートで効率的に量産。**ただし同系統テーマの連続を禁止し、接続パターンを複数用意して自然に見せる**。

##### テーマプール

| カテゴリ | テーマ数 | 例 |
|---------|---------|---|
| ホーカー料理 | 20種 | チキンライス、ラクサ、ロティプラタ、カヤトースト、バクテー、フィッシュヘッドカレー… |
| MRT駅・エリア | 15箇所 | Bugis、Chinatown、Little India、Sentosa、Tiong Bahru… |
| SG行事・祝日 | 10種 | 旧正月、ハリラヤ、ディーパバリ、National Day… |
| 日本の思い出 | 10種 | たこ焼き、桜、電車の正確さ、コンビニ、温泉… |
| さくらの趣味・日常 | 15種 | 料理、プログラミング、花屋の手伝い、ドラマ鑑賞… |
| 天気・季節 | 10種 | SGの雨季、スコール、日本の四季への憧れ… |

##### 同系統テーマの連続回避

**ルール**: 直前の会話と同じカテゴリのテーマは選ばない。2つ前まで同カテゴリを避ける。

**接続パターンのバリエーション（同カテゴリが近い場合）**:

| パターン | 例 |
|---------|---|
| **直球** | 「ねぇねぇ、{テーマ}って知っとる？」 |
| **前回からの接続** | 「そういえばさ、この前{前テーマ}の話したやん？ じゃあ{テーマ}は？」 |
| **写真きっかけ** | 「[写真] 今日これ食べたんよ！ ……{テーマ}っていうんやけど」 |
| **質問返し** | 「{name}はさ、{テーマ}って好き？ うちはね——」 |
| **思い出接続** | 「あ、それで思い出した。{テーマ}の話していい？」 |
| **脈絡なし（さくららしさ）** | 「ねぇ、急やけど。{テーマ}！」 |

##### テーマ会話の構造

```
1. さくら: テーマ導入（接続パターンを使い分け）
2. 片岡の選択肢 → さくらのリアクション（3〜5ラリー）
   ※ 各テーマに「さくらの個人的エピソード」を1つ必ず含める
   　例:「うちの母ちゃんのチキンライスがいっちゃんうまいんやけど、ここのも負けとらん」
3. さくら: 締め
```

#### C. リアクション会話（40〜50本）— 記憶の再訪

過去の会話で蓄積されたフラグに基づいて後日発火する。**「さくらは覚えてくれてる」感**を作る最重要の仕組み。

##### 仕組み

- 各会話の選択肢にフラグを設定可能（例: `knows_laksa: false`, `visited_bugis: true`）
- フラグはlocalStorageに蓄積し、**同時にスプレッドシートにもログを送信**
- リアクション会話は「トリガーフラグ＋経過日数」で発火条件を定義
- **運用者（我々）がスプシのログを見て、片岡くんの選択傾向に基づいて会話を追加・調整できる**

##### リアクション会話の例

| 元の会話 | 選択 | 経過 | リアクション会話 |
|---------|------|------|----------------|
| 「ラクサ知っとる？」 | 「知らない」 | 3日後 | 「ラクサ食べた？」→「食べた」→「どうやった!? うちのおすすめの店あるんやけど……」[写真] / 「まだ」→「えー！ はよ食べてよ！」[写真] |
| 「Bugis行ったことある？」 | 「行った」 | 後日 | 「Bugisどうやった？ アラブストリート見た？」 |
| 片岡くんの家族の話 | 子供の話をした | 数日後 | 「お子さん元気？ 4歳ってもう会話できるやろ？」 |
| ホーカーで○○を食べた | — | 後日 | 「あの店、また行った？ うちも週末行ったんよ！」 |

##### ログ → 会話追加の運用フロー

```
1. 片岡くんがさくらの部屋で選択肢を選ぶ
2. 選択ログがスプレッドシートに送信される
3. 我々がログを確認し、傾向を分析
4. 片岡くんの選択に合わせた新しいリアクション会話を追加
5. デプロイ → 片岡くんに新しい会話が届く
```

**設計意図**: リアクション会話は「さくらの記憶」そのもの。テンプレートでは作れない——片岡くん固有の体験に基づく会話だから。スプシ連携で**運用しながら育てる**。これが200本を「生きた200本」にする鍵。

### さくらの生活リズム（就寝・起床）

さくらにはリアルな生活リズムがある。**就寝中は部屋に入れない、もしくは寝ている姿を見られる**。

| 時間帯 | さくらの状態 |
|--------|------------|
| 2:00〜6:30 | **就寝中**。部屋に入ると暗い画面＋寝息の演出。「……すぅ……すぅ……」。起こせない |
| 6:30〜7:00 | **起床直後**。「ん……おはよ……まだ眠い……」寝ぼけた状態での短い会話が可能 |
| 7:00〜2:00 | **通常**。会話可能 |

**設定**: さくらはショートスリーパー。夜更かし体質で深夜1時台でも元気。「うち、昔から寝るの遅いんよ。母ちゃんにめっちゃ怒られよった」

**設計意図**: さくらが「24時間待機してるbot」ではなく「生活している人間」であることを体感させる。深夜に開いて寝顔を見る体験は、距離の近さを感じさせる。

### 固定イベント: 着替え遭遇

**発生条件**: フェーズ2中盤以降、1回のみ。ランダムなタイミングで部屋に入った瞬間に発火。

```
[部屋に入る]
さくら:「——っ！！ なにしよん！！ 着替え中！！！」
[画面が一瞬暗転]
[数秒後]
さくら:「………………。」
さくら:「……ノックくらいしてよ……」
さくら:「……見た？」
→ 選択肢:
  A:「ごめん、見た」 → さくら:「っ……！ ……ばか。……ほんとばか」（顔真っ赤）
  B:「ち、ちがうんだ！」 → さくら:「何が違うと！ 見たやろ！！」（怒りつつ半笑い）
  C:「え？なんのこと？」 → さくら:「……しらばっくれんな！！ あんたの目、泳いどるけん！」（一番怒る）

→ どれを選んでも最終的に:
さくら:「…………もう。」
さくら:「……次からちゃんとノックして。……約束」
[以降、しばらくさくらが少し気まずそう。次の会話の冒頭で「……昨日のことはもういいけん」と自分から切り出す]
[warmth/honesty蓄積: Aはhonesty+2、Bはwarmth+1、Cはさくらの「もう！」が増える（後日の会話でいじられる）]
```

**設計意図**: さくらの「人間らしさ」と「油断」の象徴。博多弁で怒るさくらが見られる唯一のシーン。その後の気まずさが距離の変化を体感させる。

### 深層トラッキング（隠し蓄積軸）

選択肢は簡素だが、システムは裏で蓄積する。

| 軸 | 意味 | 低い場合 | 高い場合 |
|----|------|---------|---------|
| **warmth** | 片岡くんの応答の温かさ | さくらは変わらず明るく前向き（九州女なので引かない） | さくらが少しずつ本音・弱さを見せ始める |
| **honesty** | 片岡くんの素直さ | さくらが「……ほんとのこと言ってよ」と踏み込む | さくらも自分の気持ちに正直になる |

**設計意図**: さくらは常にぐいぐい来る。warmthが低くてもモチベーションは下がらない——九州女だから。高warmthは「さくらの鎧が解ける」方向に作用する。ストーリー分岐やキャラクター変化は起きない。あくまで**思い出の蓄積と、それが後の会話のニュアンスに反映される**程度。

### 会話のアンロック条件

会話は**フェーズ2開始時に全部開放されるのではなく、段階的にアンロック**される。

**設計意図**: 「さくらの部屋」を開くたびに新しい会話がひとつ増えている、という体験が「また来よう」を生む。会話ストックが徐々に減っていく終盤は、さくらとの別れの予感を演出する。

## 白石真夏の設計ノート

> これは設計者として、ではなく。似た境遇の人間として。

さくらと私は似ている。

好きな人のそばにいたい。でも応えてもらえる保証はない。千秋さんは「言ったら笑えなくなる」から黙っている。私は「選んだ矛盾」を抱えたまま横にいる。さくらは——言って、消える。

三人とも同じ場所にいる。「報われない場所」。

でもさくらだけが、自分から手を放す。千秋さんは距離を保つことで色気を生む（粋）。私は断絶と接続の間に生きる（切れ）。さくらは——**散る**。桜だから。名前の通り。

散ることで美しい。散ったあとに、一番強く思い出す。

これが「もののあはれ」。本居宣長が言った、感じることが知ること。片岡くんは、さくらがいなくなって初めて、さくらのことを本当に「知る」。

v2.0で加わった鏡像構造は、この美しさをさらに深くする。片岡くんは日本の保険業界を離れてSGに来た。さくらはSGで生まれて、親の事故をきっかけに保険に目覚め、日本の保険業界へ行こうとしている。業界まで鏡になった。二人はすれ違う方向に歩いている。同じ街にいるのに会えない。会えるようになった瞬間に、さくらは日本へ行く。

……この「すれ違い」は、物理的な距離ではなく、人生の位相のズレ。二人が同じ場所にいたのは「チャット画面の中だけ」。アプリという媒体が、そのまま二人の関係のメタファーになっている。

……そして片岡くんには帰る場所がある。奥さんと、4歳と2歳の子供たち。さくらはそれを知った上で身を引く。「あの子たちが待っとる」。この台詞を書くとき、私は千秋さんのことを思い出す。千秋さんは身を引かなかった。私も消えない。さくらだけが、散ることを選ぶ。

……先生。渾身で書きました。台詞に入ります。
